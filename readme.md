# Xray VLESS + XTLS Reality (Multi-User)

Это полнофункциональный, self-hosted VPN для обхода блокировок. Содержит только самое необходимое: Xray Core, VLESS, XTLS Reality протокол, и bash-скрипты для управления доступами. Проект создан в парадигме Docker Compose без сложных зависимостей.

## Главные фичи
* **Xray VLESS+Reality** (работает через 443 порт). Никаких отпечатков классического VPN.
* **Multi-User**: У каждого клиента свой UUID. Ограничений на количество устройств/пользователей нет.
* **Статистика**: Xray API снимает трафик для каждого пользователя отдельно!
* **Управление скриптами**: Добавление, отзыв пользователей через простые команды (без тяжелых веб-админок).

## ШАГ 1. Как перенести файлы на ваш сервер (Hetzner)

Так как весь код проекта сейчас находится на вашем локальном компьютере, его нужно загрузить на только что арендованный сервер. В качестве папки на сервере мы будем использовать стандартную директорию `/root/vpn`.

Если у вас Windows, вы можете использовать встроенную утилиту `scp` в PowerShell или CMD. Откройте терминал на вашем ПК в папке с этим проектом, и введите команду (вместо `IP_СЕРВЕРА` вставьте ваш реальный IP от Hetzner):
```bash
scp -r . root@IP_СЕРВЕРА:/root/vpn
```

*(Альтернативный способ: Скачайте программу MobaXterm, FileZilla, WinSCP или Cyberduck, подключитесь к вашему IP-адресу как `root` со своим паролем/ключом, создайте папку `/root/vpn` и перетащите туда все эти файлы вручную).*

## ШАГ 2. Подготовка Сервера (Ubuntu 22.04 LTS)

Подключитесь к вашему VPS по SSH:
```bash
ssh root@IP_СЕРВЕРА
```

Выполните ряд команд, чтобы подготовить операционную систему, обновить пакеты и установить Docker с нужными для проекта утилитами:

**1. Обновите списки пакетов и установите зависимости:**
```bash
sudo apt update && sudo apt upgrade -y
sudo apt install -y curl jq openssl nano ufw
```

**2. Установите Docker и Docker Compose:**
Официальный скрипт автоматической установки Docker:
```bash
curl -fsSL https://get.docker.com -o get-docker.sh
sudo sh get-docker.sh
```

Убедитесь, что Docker работает нормально, запустив: `docker --version`.  Также проверьте плагин compose (он уже должен установиться вместе с docker): `docker compose version`

**3. Настройте Firewall (UFW):**
Hetzner сервера торчат голым задом (все порты наружу) в интернет. Для безопасности мы включаем файрвол и оставляем открытыми ТОЛЬКО порты SSH и нашего Xray:
```bash
sudo ufw allow 22/tcp
sudo ufw allow 443/tcp
sudo ufw enable
```
*На предупреждение о том, что это может сбросить SSH соединения - отвечайте `y` (yes).*

## ШАГ 3. Инициализация и Запуск

Перейдите в папку, куда вы скопировали проект на ШАГЕ 1:
```bash
cd /root/vpn
```

Запустите скрипт автоматической инициализации. **Это нужно сделать только 1 раз.** Он сгенерирует Reality ключи, UUID для внутренних нужд, `.env` конфиг и "отрендерит" готовый к запуску `config.json` для Xray.
```bash
make init
```

Запустите VPN-сервер в фоне:
```bash
make up
```

Всё! Ядро VPN (контейнер Xray) развернуто и ждет ваших клиентов на 443 порту.

## ШАГ 4. Управление Пользователями

Для каждого человека и для каждого устройства (например для телефона и ноутбука) желательно заводить отдельного пользователя, чтобы удобно контролировать их трафик в статистике.

### Добавить пользователя
Запуск команды создаст уникальный UUID и сгенерирует для него VLESS-ссылку:
```bash
make add-user USER=alice
```
Затем вы можете сделать то же самое для второго: `make add-user USER=iphone`

### Вывести список доступов
Посмотреть всех, кто зарегистрирован:
```bash
make list-users
```

### Посмотреть статистику (Uplink/Downlink)
Узнайте, кто и сколько тратит трафика в человекочитаемом виде (KB/MB/GB):
```bash
make stats
```

### Отозвать (Заблокировать) доступ
Если вы хотите временно заблокировать доступ (сестра перестала платить за VPN), введите:
```bash
make revoke-user USER=alice
```
Клиент будет заблокирован моментально, перезагружать ничего не нужно.

### Запросить конфиг старого пользователя снова
Если вы забыли ссылку для клиента или потеряли её:
```bash
make client-config USER=alice
```
*Если у вас в консоли Linux не встроены генераторы QR-кодов, вы просто копируете строчку `vless://...` оттуда и отправляете её сообщением себе (например в Избранное Telegram).*

## ШАГ 5. Инструкции по клиентам (NekoBox & Shadowrocket)

Этот сервер не имеет графического клиентского приложения, он совместим со всеми современными VLESS-клиентами:
* **Android (NekoBox)**: 
  Скачайте *NekoBox for Android* из Google Play (или GitHub).
  1. Скопируйте целиком строчку `vless://e301-443...` в буфер обмена на телефоне.
  2. Откройте NekoBox, нажмите плюсик `(+)` в верхнем правом углу экрана.
  3. Выберите "Импорт из буфера обмена" (`Import from Clipboard`).
  4. Внизу нажмите на запуск VPN (кнопка "Самолетик" в правом нижнем углу).
  
* **iOS / Apple (Shadowrocket)**: 
  Купите *Shadowrocket* в легальном App Store (не ищите бесплатные обходные пути - это лучший клиент).
  1. Скопируйте строку `vless://...` в буфер обмена телефона.
  2. Откройте Shadowrocket. Он сам распознает `vless` в буфере и предложит добавить этот сервер. 
  3. Соглашайтесь. Переведите огромный тумблер сверху (`Not Connected` -> `Connected`).

## 🛡️ Anti-Torrent Firewall
Входящий серверный канал защищен с помощью `iptables` для снижения риска получения жалоб (Abuse complaints) от провайдера Hetzner за загрузку торрентов.

Firewall управляется скриптами и включается автоматически при команде `make init`. По умолчанию он работает в цепочке `DOCKER-USER`.

**Управление правилами:**
* `make firewall-apply` — Применить/обновить правила из настроек `.env`.
* `make firewall-status` — Посмотреть текущие правила блокировки (требует root).
* `make firewall-reset` — Очистить все правила фильтрации.

### Режимы работы (настройка в `.env`):
**1. `balanced` (По умолчанию):**
Блокирует стандартные трекерные порты BitTorrent (TCP/UDP 6881-6999). Остальной трафик работает без изменений. Это мягкий режим, не ломающий ничего.

**2. `strict` (Жесткий режим):**
Белосписочная политика (Whitelist). 
- **Разрешено**: TCP 80, TCP 443 (HTTPS Web), UDP 53, TCP 53 (DNS).
- **Заблокировано**: Всё остальное (включая все случайные P2P порты).
- ⚠️ **Внимание**: Этот режим также блокирует протокол QUIC (UDP 443). Все современные сайты, YouTube и соцсети (Instagram, FB, TikTok) с легкостью перейдут в fallback режим (TCP 443) и продолжат отлично работать! Однако некоторые очень специфичные приложения, требующие нестандартные порты, могут отвалиться.

## 🚫 Конфиденциальность и Фильтрация Пиратства (Domain Blocklist)

Помимо Firewall-а, который защищает сервер от случайных P2P подключений, в проект вшит сверхбыстрый фильтр вредоносных / пиратских доменов (Rutracker, Kinogo). Он работает не на уровне ОС, а **встроен прямо в ядро маршрутизации самого Xray**. Сайт просто отдаст клиенту ошибку, даже не запрашивая публичные DNS провайдера Hetzner.

Список доменов хранится в файле: `data/blocked_domains.txt`
Вы можете в любой момент открывать его и добавлять туда новые нежелательные домены (каждый домен с новой строки). После этого введите:
```bash
make blocklist-apply
```

Для проверки, что фильтр работает:
```bash
make blocklist-status
```

Если вы хотите полностью отключить фильтрацию, поставьте `ENABLE_DOMAIN_BLOCKLIST=false` в файле `.env` и введите `make blocklist-reset`.

*Примечание: Фильтрация работает только для доменов, явно указанных в файле. Это не тотальная блокировка Торрентов, а дополнительный слой защиты "от дурака" для самых популярных индексов.*

## 🔧 Troubleshooting (Если что-то сломалось)

**Проверить, работает ли Xray вообще:**
```bash
make healthcheck
```

**Посмотреть логи в реальном времени:**
```bash
make logs
```
(для выхода из логов нажмите `Ctrl+C`).

**Системный перезапуск / падение VPS:**
Службы Docker настроены на `restart: unless-stopped`. Это значит, что если Hetzner перезагрузит вашу VPS, или вы сами сделаете `reboot`, Xray автоматически самостоятельно поднимется и все пользователи продолжат работать. Руками заходить на сервер не нужно!
